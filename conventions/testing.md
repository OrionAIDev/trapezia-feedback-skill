# Testing conventions

Applies to all skills with a `tests/` directory.

## Test structure

| Test type | When required | Location |
|---|---|---|
| Unit | Always (T1/T2) | `tests/test_<module>.py` |
| Integration | When components interact non-trivially | `tests/integration/` |
| Smoke | When deployed to OrionLab/Test | `tests/smoke/` |

Default coverage target: ≥ 80% line coverage on core modules.

## TDD discipline

Write the failing test first. Verify it fails. Implement minimally. Verify it passes.
Commit. Skipping the red step is a bug — test-first is not optional for T1/T2 skills.

## Conformance self-test

`tests/test_structure.py` (generated by `skill-template`) calls `run_audit()` from
`trapezia-skill-validator` and asserts the skill is green. This test must always pass.
Do not delete it; it is the per-skill CI gate for the conformance standard.

## Anti-patterns

- Tests that only test the validator, not the skill's own behavior ("test theater").
- Deleting regression tests because they are inconvenient.
- Hardcoding absolute paths in tests (use `tmp_path` or `skill_root` fixtures).
- Importing from `__main__` directly; use the importable module API instead.
